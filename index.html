🧠 Core Concept: Minimizing Latency via LRU Cache

When performing intensive database Lookups or Vector DB lookups, running raw network requests every time is a structural anti-pattern. An Least Recently Used (LRU) Cache discards the least recently used items first to ensure high-velocity data stays instantly readable in local memory.

▶ TIME COMPLEXITY ANALYSIS:
• Lookups (Get): O(1) Constant Time via Hash Map tracking.
• Evictions (Put): O(1) Constant Time via Doubly Linked List node re-linking.

Tip: Never back an LRU cache with a standard array/list; iterating through indices to shift elements on eviction destroys throughput by scaling linearly to O(N).
